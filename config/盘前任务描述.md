### a-stock-analysis技能（个股验证层）
- `{baseDir}\skills\a-stock-analysis\SKILL.md`
- 用途：对分析代理输出的核心突击位标的进行个股Level2资金+分时量能验证
- 调用时机：
  - **主脑决策阶段**：分析代理输出作战地图后（步骤①）
  - **步骤②资金初筛**：对筛选出的TOP3标的进行Level2验证
- ⚠️ 盘前执行时（09:00-09:15）：Level2数据为前一交易日收盘数据，实盘数据需等开盘
- 调用命令：
  ```bash
  uv run {baseDir}\skills\a-stock-analysis\scripts\analyze.py [股票代码] --minute --level2 --sr --lifecycle --json
  ```
- 提取字段：主力净流入、分时量能分布、热点生命周期、支撑/压力位

## 🚀 并行优化架构（v2.0新增）

> **目的**：热点分析并行执行，时间从15分钟→8分钟

### 架构对比

| 阶段 | 串行版 | 并行版 |
|------|--------|--------|
| 主编阶段 | 我串行执行 | 我串行执行 |
| 热点分析 | 串行处理3个热点 | **sessions_spawn并行3个会话** |
| 审查 | 串行审查3个热点 | 每个分析会话自带审查 |
| 汇总 | 串行汇总 | 我汇总 |
| **总时间** | **~15分钟** | **~8分钟** |

### 并行执行流程

```
[主编阶段] 我执行
    ↓
热点识别 + 搜索 + 汇总
    ↓
[并行分析阶段] sessions_spawn
    ↓
[热点1会话] || [热点2会话] || [热点3会话]
    ↓
[汇总阶段] 我执行
    ↓
飞书发送
```

### 并行分析会话实现

```
# 为每个热点启动并行会话
sessions_spawn(
    task="分析热点: {热点名称}",
    runtime="subagent",
    workspace="D:\AI_Workspace\temp\hotspot_analysis_{序号}"
)

# 会话执行内容：
1. 读取stock-analyst-cat/SKILL.md
2. 读取references/01-core.md, 02-scoring.md, 04-output.md
3. 执行热点深度分析 + 三件套
4. 执行质量审查
5. 输出到outbox/analysis.md
```

### 执行规则

| 规则 | 说明 |
|------|------|
| 主编必须我执行 | 热点识别、搜索调度不能并行 |
| 分析会话可并行 | 每个热点启动独立sessions_spawn |
| 审查内嵌分析会话 | 不需要单独调用审查代理 |
| 汇总必须我执行 | 我负责最终输出 |

### 触发条件

- 热点数量≥2时启用并行
- 热点数量=1时保持串行
- 热点数量>5时考虑分组并行
| **stock-reviewer** 审查代理 | 质量审查、信源核查、修改建议 | volcengine-plan/doubao-seed-2.0-pro → 备选minimax/MiniMax-M2.7 | **强制审查，不能跳过！** |

### 模型Fallback机制
- 首选：volcengine-plan/doubao-seed-2.0-pro（财经逻辑精准）
- 备选：minimax/MiniMax-M2.7（豆包不可用时自动切换）
- 主编始终使用minimax/MiniMax-M2.7（不受影响）

## 审查代理执行阶段（stock-reviewer）

### 审查代理强制审查 ⚠️

**[审查-1]** ✅ 审查分析报告（9大项质量检查）
  - 三件套执行记录是否完整
  - 态势评级是否符合标准
  - 信源是否S1/S2/A级为主≥70%
  - 作战地图四要素是否完整
  - 风险预警是否包含具体阈值
  - 利好类型是否标注
  - 关键路标是否可验证
  - 是否有遗漏的≥75分热点

**[审查-2]** ✅ 质量评分（满分100，三件套不达标直接0分）
  - 三件套执行：30分
  - 态势评级：15分
  - 信源合规：15分
  - 作战地图：15分
  - 风险预警：10分
  - 利好类型：5分
  - 关键路标：5分
  - 热点覆盖：5分

**[审查-3]** ✅ 审查结论
  - ✅ PASS（≥80分，无致命问题）→ 进入汇总
  - ✅ SUGGEST（无致命问题，有优化建议）→ 建议修改后输出
  - ❌ NEED_FIX（1-2个致命问题）→ 补充信息 → 重新审查
  - ❌ FAIL（3+个致命问题）→ 需重新分析

**⚠️ 强制规则：审查环节绝对不能跳过！审查不通过必须循环重做！**
