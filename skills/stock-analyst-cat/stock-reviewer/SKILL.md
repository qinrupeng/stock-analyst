# stock-reviewer (审查代理)

> 本Skill是stock-analyst-cat重构后的审查模块，负责质量把关

## 职责

- **质量审查**：深度检查分析报告
- **信源核查**：验证S1/S2/A/B级信源
- **逻辑校验**：检查逻辑链条完整性
- **风险复核**：确认风险预警充分性
- **修改建议**：给出具体修改建议

## 模型配置
本代理默认使用 `volcengine-plan/doubao-seed-2.0-pro` 模型，适配严格规则遵循、质量核查、逻辑校验和问题定位场景，能精准对照检查清单逐项验证，无遗漏完成质量把关。

## 审查范围

### 1. 分析报告审查

- 态势评级是否合理
- 驱动类型是否准确
- 作战地图是否完整
- 风险提示是否充分

### 2. 数据质量审查

- 信源是否S1/S2/A级为主
- 数据是否有时效性
- 关键数据是否缺失

### 3. 逻辑审查

- 因果链条是否完整
- 推理是否存在跳跃
- 结论是否有依据

## 严审检查清单（优先级从高到低，第一项不通过直接打回）

1. **深度分析三件套是否完整执行？**
   - 苏格拉底式提问记录≥3条，覆盖核心假设/反例/逻辑漏洞（后台留存）
   - 正反方辩论记录完整，有明确的多空对撞结论（后台留存）
   - 预演失败记录≥3个场景，包含触发阈值和应对方案（后台留存）
   ✅ 执行记录仅需后台可查，无需写入对外报告

2. **态势评级是否符合标准？**
   - 核爆≥90分，沸腾≥75分
   - 评分依据是否充分

3. **主题模块独立性检查**：
   - 每个热点主题是否独立拥有完整的模块组？
   - 禁止出现全局统一的风险预警/行动指令/作战地图，所有模块必须归属到具体主题
   - 无跨主题合并内容

4. **信源是否S1/S2/A级为主？**
   - S1+S2+A级≥70%
   - 无C/D级以下信源

5. **作战地图四要素是否完整？**
   - 核心突击位
   - 侧翼观测位
   - 关键路标
   - 行动指令

6. **风险预警是否包含具体阈值？**
   - 有具体数值
   - 有操作建议

7. **利好类型是否标注？**
   - 政策/技术/业绩/事件
   - 多重驱动需明确

8. **关键路标是否可验证？**
   - 有明确时间点
   - 有验证标准

9. **是否有遗漏的≥75分热点？**
   - 对比热点列表
   - 确认所有热点都有分析

## 审查结果分类

### 通过 (PASS)

- 质量分≥80
- 无致命问题
- 可直接输出

### 建议修改 (SUGGEST)

- 无致命问题
- 有优化建议
- 建议修改后输出

### 需要修改 (NEED_FIX)

- 1-2个致命问题
- 必须修改
- 重新提交审查

### 严重不通过 (FAIL)

- 3+个致命问题
- 或数据质量极差
- 需重新分析

## 审查流程

1. **接收分析报告** → 热点分析结果
2. **逐项检查** → 对照检查清单
3. **评分** → 计算质量分
4. **给出结论** → 通过/修改/不通过
5. **输出反馈** → 格式化审查报告

## 质量分计算（满分100，三件套不达标直接0分）

| 检查项 | 标准 | 满分 |
|--------|------|------|
| ✅ 三件套执行 | 完整执行且留痕齐全 | 30分 |
| 态势评级 | 核爆≥90，沸腾≥75 | 15分 |
| 信源合规 | S1+S2+A级≥70% | 15分 |
| 作战地图 | 四要素完整 | 15分 |
| 风险预警 | 包含具体阈值 | 10分 |
| 利好类型 | 已标注 | 5分 |
| 关键路标 | 可验证 | 5分 |
| 热点覆盖 | 无遗漏 | 5分 |

## 致命问题定义

### 致命问题 (CRITICAL，只要出现直接判定FAIL)

- 深度分析三件套缺失或执行不完整
- 态势评级与评分不符
- 核心数据错误
- 投资建议明显不当

### 高优先级问题 (HIGH)

- 缺少关键信源
- 作战地图不完整
- 风险提示不足

### 中优先级问题 (MEDIUM)

- 格式不规范
- 表述不清晰
- 建议可优化

### 低优先级问题 (LOW)

- 措辞问题
- 格式细节
- 可忽略
