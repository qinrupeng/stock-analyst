# 场景定义

## 场景类型

| 场景 | 关键词 | 输出要求 | 典型输入 |
|------|--------|----------|----------|
| pre_market | 盘前、开盘前、早上、早晨 | 热点预测报告 | "帮我做盘前扫描" |
| intra_day | 盘中、上午、下午、现在 | 实时热点分析 | "现在市场有什么热点" |
| after_market | 复盘、收盘、盘后、昨日 | 复盘报告 | "帮我复盘今天的市场" |
| stock_analysis | 个股、股票、600、000、002 | 个股分析报告 | "分析一下中国卫星" |

## 场景解析规则

### pre_market (盘前扫描)

- **目标**：预测今日热点，提前布局
- **重点**：
  - 隔夜外盘走势
  - 夜盘期货表现
  - 政策/消息面
  - 机构晨会观点
- **输出**：今日潜力热点排名 + 推荐标的

### intra_day (盘中分析)

- **目标**：实时跟踪热点切换
- **重点**：
  - 当前热点板块
  - 资金流向
  - 涨跌幅榜
  - 异动个股
- **输出**：实时热点跟踪 + 操作建议

### after_market (收盘复盘)

- **目标**：总结今日市场表现
- **重点**：
  - 今日涨跌停分析
  - 板块轮动回顾
  - 资金流向统计
  - 明日展望
- **输出**：复盘报告 + 明日计划

### stock_analysis (个股分析)

- **目标**：深度分析特定股票
- **重点**：
  - 基本面分析
  - 技术面分析
  - 行业地位
  - 估值分析
- **输出**：个股深度分析报告

## 场景识别示例

| 用户输入 | 识别场景 | 置信度 |
|----------|----------|--------|
| "今天有啥热点" | intra_day | 高 |
| "早上好帮我看看盘面" | pre_market | 高 |
| "复盘一下昨天" | after_market | 高 |
| "600118怎么样" | stock_analysis | 高 |
| "帮我分析下这个股票" | stock_analysis | 中 |
| "现在能买什么" | intra_day | 中 |

## 场景切换

在分析过程中可能需要场景切换：

- 盘前扫描 → 发现个股机会 → 切换到个股分析
- 个股分析 → 用户问今日走势 → 切换到盘中分析

切换时需要：
1. 保存当前上下文
2. 识别新场景
3. 调整分析重点
4. 合并输出结果
