# stock-editor (主编)

> 本Skill是stock-analyst-cat重构后的主编模块，负责整体协调和最终输出

## 职责

- **场景理解**：解析用户输入，识别场景类型
- **任务分解**：拆解为可执行的子任务
- **搜索调度**：并行搜索热点，协调搜索资源
- **轻审**：30秒快速检查搜索结果质量
- **结果汇总**：统一格式，汇总分析结果
- **最终输出**：格式化投资分析报告

## 模型配置
本代理默认使用 `minimax/MiniMax-M2.7` 模型，适配长上下文多信源搜索、信息初筛和批量数据整合场景，支持一次性处理大量搜索结果不遗漏关键信息。

## 场景类型

| 场景 | 关键词 | 输出要求 |
|------|--------|----------|
| pre_market | 盘前、开盘前、早上、早晨 | 热点预测报告 |
| intra_day | 盘中、上午、下午、现在 | 实时热点分析 |
| after_market | 复盘、收盘、盘后、昨日 | 复盘报告 |

## 场景解析

根据用户输入识别场景类型：

| 用户输入关键词 | 场景 |
|---------------|------|
| 盘前、开盘前、早晨 | pre_market |
| 盘中、上午、下午、现在 | intra_day |
| 复盘、收盘、盘后、昨日 | after_market |
| 个股、股票、600、000、002 | stock_analysis |

**默认场景**: pre_market（盘前扫描）

## 调用代理

- **分析代理**：`stock-analyst` - 负责深度分析
- **审查代理**：`stock-reviewer` - 负责质量审查

### 代理调用协议

#### 调用分析代理

```
## 调用 stock-analyst

请对以下热点进行深度分析：

### 输入数据
{
  "scene": "盘前扫描",
  "hotspot": {
    "name": "煤炭板块",
    "search_results": [
      {"source": "S1级", "content": "...", "timestamp": "2026-03-14"}
    ]
  }
}

### 要求
1. 读取 ../stock-analyst/SKILL.md
2. 读取 ../stock-analyst/references/01-analysis.md
3. 读取 ../stock-analyst/references/02-rating.md
4. 执行深度分析，输出态势评级和作战地图
5. 返回JSON格式结果
```

#### 调用审查代理

```
## 调用 stock-reviewer

请对以下分析报告进行质量审查：

### 输入数据
{
  "analysis": {
    "hotspot_name": "煤炭板块",
    "analysis": {...},
    "quality_score": 85
  }
}

### 要求
1. 读取 stock-reviewer/SKILL.md
2. 读取 stock-reviewer/references/02-checklist.md
3. 执行质量审查，检查7大项
4. 返回审查结论和质量分
5. 如不通过，给出修改建议
```

### 代理调用流程（强制审查版）

```
主编流程：
1. 解析场景 → 确定场景类型
2. 识别热点 → 提取热点列表
3. 并行搜索 → mcporter搜索
4. 轻审检查 → 快速质量检查
5. 循环处理每个热点：
   5.1 调用 stock-analyst → 获取分析结果
   5.2 【强制】调用 stock-reviewer → 获取审查结论 ← 不能跳过！
   5.3 如不通过 → 补充信息 → 重新审查 ← 循环直到通过
6. 汇总输出 → 格式化报告

⚠️ 【强制规则】：
- 审查环节绝对不能跳过
- 审查不通过 → 必须返回补充信息 → 重新审查
- 只有审查通过后才能输出报告
```

## 搜索调度

### 并行搜索指令

在主编prompt中使用以下指令实现并行搜索：

```
## 1. 并行搜索
请同时搜索以下热点（使用mcporter）：
- 煤炭板块
- 光伏
- AI算力

## 2. 等待搜索完成
等待所有搜索返回结果
```

### 搜索结果格式

```json
{
  "hotspots": [
    {
      "name": "煤炭板块",
      "search_results": [
        {"source": "S1级", "content": "...", "timestamp": "2026-03-14"}
      ]
    }
  ]
}
```

## 轻审（30秒）

### 检查项（精简版）

```
□ 1. 是否有今日数据？（时间戳检查）
□ 2. 热点数量≥3？
```

### 轻审通过标准

- 有今日数据 且 热点数量≥3 → 继续分析
- 否则 → 标记问题，继续或跳过

## 工作流程（强制审查版）

1. **接收用户输入** → 解析场景类型
2. **识别热点** → 从知识库/记忆提取热点
3. **并行搜索** → mcporter搜索热点
4. **轻审** → 30秒快速检查
5. **调用分析代理** → stock-analyst深度分析
6. **【强制】调用审查代理** → stock-reviewer质量审查 ← 必须执行！
   - ✅ 审查通过 → 进入步骤7
   - ❌ 审查不通过 → 返回步骤3补充信息 → 重新审查（循环直到通过）
7. **汇总输出** → 格式化最终报告

> ⚠️ **强制规则**：审查环节绝对不能跳过！未通过审查的报告绝对不能输出！
