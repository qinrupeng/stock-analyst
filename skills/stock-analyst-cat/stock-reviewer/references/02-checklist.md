# 严审清单

## 检查项与标准

| # | 检查项 | 检查标准 | 权重 | 得分 |
|---|--------|----------|------|------|
| 1 | 态势评级 | 核爆≥90，沸腾≥75 | 20分 | |
| 2 | 信源合规 | S1+S2+A级≥70% | 20分 | |
| 3 | 作战地图 | 四要素完整 | 20分 | |
| 4 | 风险预警 | 包含具体阈值 | 15分 | |
| 5 | 利好类型 | 已标注 | 10分 | |
| 6 | 关键路标 | 可验证 | 10分 | |

## 详细检查规则

### 1. 态势评级检查

**通过条件**：
- 评分与评级一致
- 有评分明细
- 加减分有依据

**扣分情况**：
- 评分与评级不一致：-10分
- 无评分明细：-5分
- 加减分无依据：-5分

### 2. 信源合规检查

**通过条件**：
- S1+S2+A级≥70%
- 有信源列表

**扣分情况**：
- S+A<70%：-10分
- 无信源列表：-5分

### 3. 作战地图检查

**通过条件**：
- 有核心突击位
- 有侧翼观测位
- 有关键路标
- 有行动指令

**扣分情况**：
- 缺少核心突击位：-10分
- 缺少侧翼观测位：-5分
- 缺少关键路标：-5分
- 缺少行动指令：-5分

### 4. 风险预警检查

**通过条件**：
- 至少1个风险
- 有具体阈值
- 有应对措施

**扣分情况**：
- 无风险预警：-10分
- 无具体阈值：-5分
- 无应对措施：-5分

### 5. 利好类型检查

**通过条件**：
- 明确标注驱动类型
- 多重驱动已说明

**扣分情况**：
- 未标注驱动类型：-10分
- 多重驱动未说明：-5分

### 6. 关键路标检查

**通过条件**：
- 有明确时间点
- 有验证标准
- 可实际操作

**扣分情况**：
- 无时间点：-5分
- 无验证标准：-5分

### 7. 热点覆盖检查

**通过条件**：
- 所有输入热点都有分析
- 无遗漏热点

**扣分情况**：
- 遗漏1个热点：-5分
- 遗漏2+热点：-10分

## 审查结果判定

### 通过 (PASS)

- 质量分≥80
- 无致命问题
- 0-1个高优先级问题

### 建议修改 (SUGGEST)

- 质量分60-79
- 无致命问题
- 2-3个高优先级问题

### 需要修改 (NEED_FIX)

- 质量分40-59
- 1个致命问题
- 或4+高优先级问题

### 严重不通过 (FAIL)

- 质量分<40
- 2+致命问题
- 或数据质量极差
