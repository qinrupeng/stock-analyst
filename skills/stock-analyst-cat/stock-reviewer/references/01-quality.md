# 质量标准

**文档版本**：1.1（2026-03-23 更新：删除D级信源定义，与06-sources.md五级架构保持一致）

## 分析报告质量体系

### 五维质量评估

| 维度 | 权重 | 说明 |
|------|------|------|
| 信息质量 | 30% | 信源可靠性、数据准确性 |
| 逻辑质量 | 25% | 推理严密性、结论可靠性 |
| 实用质量 | 20% | 可操作性、投资建议 |
| 时效质量 | 15% | 数据时效性、分析及时性 |
| 呈现质量 | 10% | 格式规范、表述清晰 |

### 信息质量标准

#### 信源分级

- **S1级**：官方机构（.gov.cn）
- **S2级**：权威媒体（新华社、人民日报、央视等）
- **A级**：主流财经媒体、行业专家观点
- **B级**：普通媒体报道、二手信息
- **C级**：数据服务商（仅数据引用）

#### 信源要求

- S1+S2+A级≥70%
- C级信源需标注"仅参考"

#### 时效性要求

> **说明**：本文件的时效性要求是**分析质量标准**，关注数据的新鲜度要求。信源的时效性标注规则详见 `06-sources.md` 的时效性标注章节，两者配套使用。

- 盘中分析：数据≤30分钟
- 盘后复盘：数据≤2小时
- 盘前预测：数据≤8小时

> **信源时效性标注对照**（详见06-sources.md）：
> - 🆕：0-6小时（刚刚发布）
> - 📅：6-24小时（今日消息）
> - ⏰：24-72小时（隔日仍有价值）
> - ❌：>72小时（过时消息）

### 逻辑质量标准

#### 因果链条

```
驱动因素 → 逻辑推导 → 结论
```

每个环节必须有据可依，不能跳跃。

#### 逻辑检查点

- 驱动因素是否真实？
- 推导过程是否合理？
- 结论是否充分？
- 是否有反例？

### 实用质量标准

#### 投资建议要素

| 要素 | 说明 |
|------|------|
| 标的 | 具体股票代码 |
| 仓位 | 建议比例 |
| 时机 | 买入时机 |
| 目标 | 预期涨幅 |
| 止损 | 止损位置 |

#### 风险提示要素

| 要素 | 说明 |
|------|------|
| 风险类型 | 具体风险 |
| 触发条件 | 何时触发 |
| 阈值 | 具体数值 |
| 应对措施 | 如何处理 |

### 呈现质量标准

#### 格式要求

- 关键信息突出显示
- 数据用表格呈现
- 逻辑用流程图（可选）

#### 表述要求

- 专业但不晦涩
- 准确但不啰嗦
- 自信但不夸大
