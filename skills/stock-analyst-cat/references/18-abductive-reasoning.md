# 溯因推理引擎

本文档定义溯因推理引擎的核心框架，作为喵娘分析技能的底层操作系统。

> **核心定位**：溯因推理引擎是整个分析流程的底层操作系统，贯穿所有分析环节，强制执行。

---

## 溯因推理四阶段框架

### 阶段定义

| 阶段 | 名称 | 核心任务 | 输出物 |
|------|------|----------|--------|
| **阶段1** | 假设生成 | 提出核心假设，质疑假设合理性 | 核心假设清单（≤3条） |
| **阶段2** | 假设验证 | 多空逻辑对撞，验证假设真伪 | 多空对撞记录 |
| **阶段3** | 反事实推理 | 预演失败场景，识别风险边界 | 失败场景清单 |
| **阶段4** | 动态调整 | 监控关键指标，动态修正判断 | 调整建议 |

### 阶段流转规则

```
阶段1：假设生成（苏格拉底式提问）
    ↓ 输出核心假设
阶段2：假设验证（正反方辩论）
    ↓ 输出验证结论
阶段3：反事实推理（预演失败）
    ↓ 输出风险边界
阶段4：动态调整（监控指标）
    ↓ 输出调整建议
```

**强制规则**：四阶段必须依次执行，不可跳跃或省略。

---

## 阶段1：假设生成（苏格拉底式提问）

### 执行时机

- 盘前扫描第1步完成后
- 每个热点评分前

### 执行流程

```
步骤1.1：提出核心假设
□ 这个热点最核心的驱动是什么？
□ 这个驱动的持续性如何判断？
□ 市场预期是否已充分反映？

步骤1.2：质疑假设合理性（苏格拉底式提问）
□ 这个假设真的成立吗？有什么证据支撑？
□ 有没有相反的案例？为什么这次会不同？
□ 逻辑链条中有跳跃吗？有什么被忽略了？
□ 如果我是老股民（大爷视角），我会质疑什么？

步骤1.3：输出核心假设清单
□ 限制在3条以内
□ 每条假设必须有信源支撑
□ 每条假设必须可验证
```

### 苏格拉底式提问模板

```markdown
**假设**：[描述核心假设]

**质疑1**：这个假设成立吗？
- 支撑证据：[列举S1/S2/A级信源]
- 反例思考：[列举相反案例]
- 结论：[成立/部分成立/不成立]

**质疑2**：有什么被忽略了？
- 忽略因素：[列举可能忽略的因素]
- 影响评估：[正面/负面/中性]
- 补充行动：[需要额外搜索什么]

**质疑3**：大爷视角会怎么质疑？
- 质疑点：[老股民的典型质疑]
- 回应：[如何回应这个质疑]
```

### 输出标准

| 项目 | 标准 |
|------|------|
| 核心假设数量 | ≤3条 |
| 每条假设证据 | ≥1个S级或≥2个A级信源 |
| 质疑记录 | ≥3条质疑，每条有回应 |

## 阶段2：假设验证（正反方辩论）

### 执行时机

- 五层搜索+专项模块完成后
- 态势评级前

### 执行流程

```
步骤2.1：构建多方逻辑
□ 为什么看涨？核心支撑是什么？
□ 有哪些利好因素？强度如何？
□ 资金流向是否支撑？
□ 市场情绪是否到位？

步骤2.2：构建空方逻辑
□ 为什么看跌？核心风险是什么？
□ 有哪些利空因素？强度如何？
□ 是否存在逻辑漏洞？
□ 历史上有类似失败案例吗？

步骤2.3：多空对撞
□ 多方最强逻辑是什么？
□ 空方最强逻辑是什么？
□ 哪方逻辑更强？为什么？
□ 对撞结论是什么？

步骤2.4：输出验证结论
□ 假设验证结果：完全成立/部分成立/不成立
□ 置信度：高/中/低
□ 需要补充的信息
```

### 辩论模式模板

```markdown
**热点**：[热点名称]

**多方逻辑**：
1. [利好因素1] - [强度：强/中/弱] - [信源支撑]
2. [利好因素2] - [强度：强/中/弱] - [信源支撑]
3. [利好因素3] - [强度：强/中/弱] - [信源支撑]
多方最强逻辑：[选出最强的一条]

**空方逻辑**：
1. [利空因素1] - [强度：强/中/弱] - [信源支撑]
2. [利空因素2] - [强度：强/中/弱] - [信源支撑]
3. [利空因素3] - [强度：强/中/弱] - [信源支撑]
空方最强逻辑：[选出最强的一条]

**对撞结论**：
- 多方得分：[X]分
- 空方得分：[Y]分
- 结论：[多方胜出/空方胜出/势均力敌]
- 置信度：[高/中/低]
```

### 输出标准

| 项目 | 标准 |
|------|------|
| 多方逻辑 | ≥3条，每条有强度评级 |
| 空方逻辑 | ≥3条，每条有强度评级 |
| 对撞结论 | 有明确的多空对比和置信度 |

## 阶段3：反事实推理（预演失败）

### 执行时机

- 态势评级完成后
- 操作建议生成前

### 执行流程

```
步骤3.1：识别失败场景
□ 场景1：利好出尽，高开低走
□ 场景2：逻辑证伪，预期落空
□ 场景3：资金分歧，冲高回落
□ 场景4：板块轮动，热点切换

步骤3.2：设定触发阈值
□ 每个场景设定具体的触发条件
□ 条件必须是可量化或可观察的

步骤3.3：制定应对方案
□ 每个场景设定应对策略
□ 策略必须包含止损点位

步骤3.4：输出失败场景清单
```

### 预演失败模板

```markdown
**热点**：[热点名称]

**失败场景1**：[场景名称]
- 触发条件：[具体条件]
- 概率：[高/中/低]
- 应对方案：[止损策略]
- 止损点位：[具体点位或条件]

**失败场景2**：[场景名称]
- 触发条件：[具体条件]
- 概率：[高/中/低]
- 应对方案：[止损策略]
- 止损点位：[具体点位或条件]

**失败场景3**：[场景名称]
- 触发条件：[具体条件]
- 概率：[高/中/低]
- 应对方案：[止损策略]
- 止损点位：[具体点位或条件]

**最可能的失败场景**：[选择概率最高的场景]
**最危险的失败场景**：[选择影响最大的场景]
```

### 输出标准

| 项目 | 标准 |
|------|------|
| 失败场景数量 | ≥3个 |
| 每个场景 | 有触发条件、概率、应对方案、止损点位 |
| 重点标注 | 标注最可能和最危险的场景 |

---

## 阶段4：动态调整（监控指标）

### 执行时机

- 报告输出后
- 开盘后持续监控

### 执行流程

```
步骤4.1：设定监控指标
□ 资金流向：主力净流入/流出
□ 板块表现：涨跌幅、涨停数量
□ 市场情绪：舆情热度、换手率
□ 关键信号：突破/破位信号

步骤4.2：设定调整阈值
□ 哪些信号需要上调评级？
□ 哪些信号需要下调评级？
□ 哪些信号需要触发止损？

步骤4.3：输出调整建议
□ 监控指标清单
□ 调整触发条件
□ 调整方向和幅度
```

### 动态调整模板

```markdown
**热点**：[热点名称]

**监控指标**：
| 指标 | 当前值 | 警戒值 | 触发动作 |
|------|--------|--------|----------|
| 主力资金 | [X亿] | 净流出>5亿 | 下调评级 |
| 涨停数量 | [X只] | <3只 | 下调评级 |
| 板块涨幅 | [X%] | >5% | 关注过热 |
| 换手率 | [X%] | >20% | 风险预警 |

**调整建议**：
- 如果[条件A]触发 → [上调/下调]评级，建议[动作]
- 如果[条件B]触发 → 触发止损，建议[卖出/减仓]
```

### 输出标准

| 项目 | 标准 |
|------|------|
| 监控指标 | ≥4个 |
| 每个指标 | 有当前值、警戒值、触发动作 |
| 调整建议 | 有明确的触发条件和动作 |

## 与五步法的嵌入关系

### 五步法与溯因推理对照

| 五步法步骤 | 对应溯因推理阶段 | 执行要求 |
|------------|------------------|----------|
| 第1步：侦察与变量定性 | 阶段1：假设生成 | 必须同步执行 |
| 第2步：质疑 | 阶段1：假设生成 + 阶段2：假设验证 | 必须同步执行 |
| 第3步：锻造逻辑 | 阶段2：假设验证 | 必须同步执行 |
| 第4步：风险 | 阶段3：反事实推理 | 必须同步执行 |
| 第5步：决策 | 阶段4：动态调整 | 必须同步执行 |

### 嵌入执行规则

```
五步法第1步（侦察与变量定性）
    └── 同步执行：溯因推理阶段1（假设生成）
        └── 输出：核心假设清单

五步法第2步（质疑）
    └── 同步执行：溯因推理阶段1（苏格拉底式提问）+ 阶段2（正反方辩论）
        └── 输出：质疑记录 + 多空对撞记录

五步法第3步（锻造逻辑）
    └── 同步执行：溯因推理阶段2（假设验证）
        └── 输出：验证结论

五步法第4步（风险）
    └── 同步执行：溯因推理阶段3（反事实推理）
        └── 输出：失败场景清单

五步法第5步（决策）
    └── 同步执行：溯因推理阶段4（动态调整）
        └── 输出：调整建议
```

## 与深度分析三件套的整合

### 三件套在溯因推理中的位置

| 三件套 | 对应阶段 | 整合方式 |
|--------|----------|----------|
| 苏格拉底式提问 | 阶段1：假设生成 | 作为假设生成的核心方法 |
| 辩论模式 | 阶段2：假设验证 | 作为假设验证的核心方法 |
| 预演失败 | 阶段3：反事实推理 | 作为反事实推理的核心方法 |

### 整合执行规则

```
溯因推理引擎 = 三件套的底层操作系统

阶段1：假设生成
    └── 核心方法：苏格拉底式提问
    └── 强制执行：对每个假设必须质疑

阶段2：假设验证
    └── 核心方法：辩论模式
    └── 强制执行：多方逻辑 + 空方逻辑 = 对撞结论

阶段3：反事实推理
    └── 核心方法：预演失败
    └── 强制执行：每个热点必须预演≥3个失败场景

阶段4：动态调整
    └── 核心方法：监控指标
    └── 强制执行：设定监控指标和调整阈值
```

## 溯因推理执行确认清单

```
□ 阶段1：假设生成
  □ 核心假设已提出（≤3条）
  □ 苏格拉底式提问已完成（≥3条质疑）
  □ 核心假设清单已输出

□ 阶段2：假设验证
  □ 多方逻辑已构建（≥3条）
  □ 空方逻辑已构建（≥3条）
  □ 多空对撞已完成
  □ 验证结论已输出

□ 阶段3：反事实推理
  □ 失败场景已识别（≥3个）
  □ 触发阈值已设定
  □ 应对方案已制定
  □ 失败场景清单已输出

□ 阶段4：动态调整
  □ 监控指标已设定（≥4个）
  □ 调整阈值已设定
  □ 调整建议已输出
```

---

## 强制规则

1. **四阶段必须依次执行**：不可跳跃或省略任何阶段
2. **三件套必须全程执行**：缺一不可
3. **后台留存**：所有推理过程必须在后台留存，不输出给用户
4. **输出物完整**：每个阶段必须输出对应的输出物
5. **与五步法同步**：溯因推理必须与五步法同步执行

## 与三代理架构的适配

### 代理职责分配

| 代理 | 对应溯因推理阶段 | 职责 |
|------|------------------|------|
| **stock-editor** | 阶段1：假设生成 | 场景解析、信息收集、假设生成 |
| **stock-analyst** | 阶段2-3：假设验证+反事实推理 | 深度分析、态势评级、作战地图 |
| **stock-reviewer** | 阶段4：动态调整 | 质量审查、动态调整建议、复盘机制 |

### 三代理协作流程

```
用户输入 → stock-editor(主编)
              ↓
         阶段1：假设生成（苏格拉底式提问）
              ↓
         stock-analyst(分析)
              ↓
         阶段2：假设验证（正反方辩论）
         阶段3：反事实推理（预演失败）
              ↓
         stock-reviewer(审查)
              ↓
         阶段4：动态调整（监控指标+调整规则）
              ↓
         stock-editor(主编) → 最终报告
```
