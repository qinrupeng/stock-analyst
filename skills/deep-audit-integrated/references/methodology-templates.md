## 矛盾模板

### 模板1：唯一性矛盾
**提问格式**："为什么只有[现象A]出现了，而[相似场景B]没有？"

**应用场景**：
- 特斯拉销量下滑 vs 苹果销量稳定
- 某项目成功 vs 类似项目失败
- 某地区爆发疫情 vs 其他地区未受影响

**生成假设方向**：
寻找A特有因素（供应链、定价、竞争、管理、环境等）

**示例**：
```
[现象] 为什么只有特斯拉Q3销量下滑，而苹果同期销量稳定？
[疑问] 同样面临供应链问题，为什么只有特斯拉受影响？
[假设方向] 特斯拉特有的供应链依赖度、产品生命周期、市场竞争格局
```

---

### 模板2：反向矛盾
**提问格式**："如果[假设H1]成立，为什么[预期结果]没有出现？"

**应用场景**：
- 如果供应链是主因，为什么高价车型销量未受影响？
- 如果降价有效，为什么销量没有回升？
- 如果营销有效，为什么转化率没有提升？

**生成假设方向**：
质疑H1的充分性，寻找备选解释

**示例**：
```
[现象] 特斯拉降价后销量仍下滑
[疑问] 如果价格是主要因素，为什么降价没有带来销量回升？
[假设方向] 可能存在其他更重要的因素：品牌形象、产品竞争力、市场饱和
```

---

### 模板3：时间矛盾
**提问格式**："为什么[现象]在[时间点A]出现，而在[时间点B]没有？"

**应用场景**：
- 为什么Q3销量下滑，而Q4恢复？
- 为什么去年策略有效，今年失效？
- 为什么早高峰拥堵，晚高峰不拥堵？

**生成假设方向**：
短期因素（季节性、促销、事件）vs 长期因素（战略调整、市场变化、技术迭代）

**示例**：
```
[现象] 为什么特斯拉Q3销量下滑，而Q4恢复？
[疑问] 是短期波动还是长期趋势的转折点？
[假设方向] Q3可能有特殊事件（供应链中断、竞争加剧），Q4有促销或新产品发布
```

---

### 模板4：边界矛盾
**提问格式**："为什么[现象]在[边界条件]下不成立？"

**应用场景**：
- 为什么策略在测试环境有效，但生产环境失效？
- 为什么在中小客户群体有效，大客户无效？
- 为什么国内市场有效，海外市场无效？

**生成假设方向**：
寻找边界特异性因素（规模、环境、资源、文化）

**示例**：
```
[现象] 为什么策略在测试环境有效，但生产环境失效？
[疑问] 测试环境和生产环境的关键差异是什么？
[假设方向] 数据规模、用户行为、系统负载、外部干扰
```

---

### 模板5：空间矛盾
**提问格式**："为什么[现象]在A地出现，B地没有？"

**应用场景**：
- 为什么在一线城市用户增长快，二三线城市慢？
- 为什么南方市场接受度高，北方市场低？
- 为什么总部执行力强，分公司执行力弱？

**生成假设方向**：
区域特异性因素（经济水平、竞争格局、文化差异、政策环境）

**示例**：
```
[现象] 为什么在一线城市用户增长快，二三线城市慢？
[疑问] 不同城市用户的需求差异是什么？
[假设方向] 收入水平、消费习惯、品牌认知、渠道覆盖
```

---

### 模板6：群体矛盾
**提问格式**："为什么[现象]在群体A中出现，群体B没有？"

**应用场景**：
- 为什么年轻用户喜欢，年长用户不喜欢？
- 为什么付费用户留存高，免费用户留存低？
- 为什么新员工绩效好，老员工绩效差？

**生成假设方向**：
群体特异性因素（需求差异、使用习惯、价值认知、激励机制）

**示例**：
```
[现象] 为什么年轻用户喜欢，年长用户不喜欢？
[疑问] 不同年龄群体的核心需求是什么？
[假设方向] 产品设计偏好、技术接受度、时间分配、社交需求
```

## 假设竞争矩阵

### 评分标准详解

#### 解释力 (0-10)
评估假设对观测现象的解释能力：

| 分数 | 标准 | 说明 |
|------|------|------|
| 9-10 | 极强 | 能解释90%以上观测点，覆盖所有关键现象 |
| 7-8 | 强 | 能解释70-90%观测点，覆盖主要现象 |
| 5-6 | 中等 | 能解释50-70%观测点，覆盖部分现象 |
| 3-4 | 弱 | 能解释30-50%观测点，解释力较弱 |
| 0-2 | 极弱 | 能解释30%以下观测点，解释力极弱 |

**示例**：
```
假设H1：供应链问题导致销量下滑
解释力评分：8.5
理由：能解释为什么销量下滑、为什么特定车型受影响更大、为什么Q3最严重
但无法完全解释为什么高价车型销量稳定
```

---

#### 可验证性 (0-10)
评估假设验证的可行性和成本：

| 分数 | 标准 | 说明 |
|------|------|------|
| 9-10 | 极高 | 有明确数据来源，可快速验证 |
| 7-8 | 高 | 有验证路径，需要一定成本 |
| 5-6 | 中等 | 验证路径模糊，需要创造性方法 |
| 3-4 | 低 | 难以验证，缺乏直接数据 |
| 0-2 | 极低 | 几乎无法验证 |

**示例**：
```
假设H1：供应链问题导致销量下滑
可验证性评分：9.0
理由：可通过生产数据、交付周期、库存水平等直接数据验证
验证方法：对比Q2/Q3/Q4的供应链指标与销量数据
```

---

#### 独特性 (0-10)
评估假设与其他假设的差异化程度：

| 分数 | 标准 | 说明 |
|------|------|------|
| 9-10 | 极高 | 与其他假设完全不同，提供独特视角 |
| 7-8 | 高 | 与其他假设有明显差异 |
| 5-6 | 中等 | 与其他假设部分重叠 |
| 3-4 | 低 | 与其他假设高度相似 |
| 0-2 | 极低 | 与其他假设几乎相同 |

**示例**：
```
假设H1：供应链问题
假设H2：定价策略失误
假设H3：品牌形象受损

独特性评分：
H1: 7.5 - 与H2部分相关（定价受供应链成本影响），但核心机制不同
H2: 8.0 - 与H1有部分关联，但侧重市场策略而非运营问题
H3: 9.0 - 完全不同的视角，关注长期品牌价值而非短期运营
```

---

#### 简洁性 (0-10)
评估假设的复杂度和奥卡姆剃刀原则符合度：

| 分数 | 标准 | 说明 |
|------|------|------|
| 9-10 | 极简 | 简单明了，符合奥卡姆剃刀原则 |
| 7-8 | 简洁 | 相对简洁，逻辑清晰 |
| 5-6 | 中等 | 有一定复杂性，但可理解 |
| 3-4 | 复杂 | 复杂，需要多个前提 |
| 0-2 | 极复杂 | 极度复杂，难以理解 |

**示例**：
```
假设H1：供应链问题导致销量下滑
简洁性评分：9.0
理由：单一因果链，无需复杂前提
供应链问题 → 产能下降 → 交付延迟 → 销量下滑

假设H3：品牌形象受损
简洁性评分：7.0
理由：需要解释品牌形象如何受损、如何影响购买决策、为什么影响Q3而非Q4
```

---

### 综合评分计算

**公式**：
```
综合评分 = (解释力 × 0.4) + (可验证性 × 0.3) + (独特性 × 0.2) + (简洁性 × 0.1)
```

**权重说明**：
- 解释力最重要（40%）：假设的核心价值在于解释现象
- 可验证性次之（30%）：无法验证的假设没有实用价值
- 独特性中等（20%）：提供多角度分析
- 简洁性辅助（10%）：优先简洁但不强求

## 反事实推演模板

### 模板1：因素移除推演

**结构**：
```
[反事实推演] 因素移除推演
原场景：[描述当前场景]
移除因素：[X因素]
推演结果：[如果移除X，Y会发生什么变化？]
结论：[X是必要条件/充分条件/既非必要也非充分条件]
```

**应用场景**：
- 如果移除某个功能，用户行为会如何变化？
- 如果没有某个资源，项目能否成功？
- 如果不进行某项投资，会有什么结果？

**示例**：
```
[反事实推演] 因素移除推演
原场景：特斯拉Q3销量下滑，存在供应链问题
移除因素：供应链问题
推演结果：如果供应链正常，销量可能回升15-20%，但仍低于Q1水平
结论：供应链问题是必要条件，但非充分条件
启示：需要同时解决定价和竞争问题
```

---

### 模板2：因果链推演

**结构**：
```
[反事实推演] 因果链推演
假设：[X假设]
一级效应：[直接影响]
二级效应：[间接影响]
三级效应：[长期影响]
观测验证：[哪些效应已观测到？哪些未观测到？]
结论：[假设成立/部分成立/不成立]
```

**应用场景**：
- 某项政策实施后的连锁反应
- 技术升级的影响路径
- 组织变革的长期效果

**示例**：
```
[反事实推演] 因果链推演
假设：供应链问题导致销量下滑
一级效应：产能下降20%，交付延迟增加15天
二级效应：用户等待时间变长，部分用户转向竞品
三级效应：品牌口碑受损，长期客户流失
观测验证：
  - 一级效应已观测：产能和交付数据确认
  - 二级效应部分观测：竞品销量上升10%，但转向比例不明确
  - 三级效应未观测：需Q4数据验证
结论：假设部分成立，需要更多数据验证二三级效应
```

---

### 模板3：极端边界推演

**结构**：
```
[反事实推演] 极端边界推演
原场景：[描述当前场景]
极端值：[X达到极端值的情况]
推演结果：[Y会发生什么极端变化？]
临界点：[X在什么值时Y开始失控？]
应对策略：[如何避免极端情况？]
```

**应用场景**：
- 市场份额极端下降的风险
- 成本极端上升的应对
- 用户极端流失的预案

**示例**：
```
[反事实推演] 极端边界推演
原场景：用户增长策略依赖裂变机制
极端值：裂变系数从1.5降至1.0（无增长）
推演结果：用户增长停滞，6个月内用户基数下降30%
临界点：裂变系数低于1.2时，增长无法覆盖流失
应对策略：
  1. 建立裂变系数实时监控，低于1.3时预警
  2. 准备付费获客预算，裂变失效时快速启用
  3. 优化留存策略，降低用户流失率
```

---

### 模板4：时间反转推演

**结构**：
```
[反事实推演] 时间反转推演
原场景：[描述当前场景]
时间倒流到：[某个关键时间点]
改变决策：[如果那时做了不同的选择]
推演结果：[现在的结果会有什么不同？]
结论：[时序因果是否成立？]
```

**应用场景**：
- 评估历史决策的影响
- 识别关键转折点
- 为未来决策提供参考

**示例**：
```
[反事实推演] 时间反转推演
原场景：Q3销量下滑，Q4恢复
时间倒流到：Q2末（销量下滑前）
改变决策：提前囤积关键零部件，避免供应链中断
推演结果：
  - Q3销量可能仅下降5%而非20%
  - 不需要大幅降价促销
  - 品牌形象未受影响
结论：供应链管理是关键转折点，时序因果成立
启示：建立供应链预警机制，提前应对风险
```

---

### 模板5：资源约束推演

**结构**：
```
[反事实推演] 资源约束推演
原场景：[描述当前场景]
资源约束：[预算/人力/时间减半]
推演结果：[结果会如何变化？]
关键依赖：[哪些环节最依赖资源？]
优化策略：[如何用更少资源达成目标？]
```

**应用场景**：
- 预算削减的应对方案
- 人力不足的优先级调整
- 时间紧迫的策略优化

**示例**：
```
[反事实推演] 资源约束推演
原场景：用户增长策略需要100万预算
资源约束：预算减半至50万
推演结果：
  - 付费获客规模下降50%
  - 需要更依赖裂变和内容营销
  - 增长速度可能下降30-40%
关键依赖：
  - 付费渠道效率（CPA最低的渠道优先）
  - 裂变机制效果（需要优化激励设计）
优化策略：
  1. 砍掉低效渠道，聚焦高ROI渠道
  2. 增加裂变激励，降低获客成本
  3. 强化内容营销，提升自然流量
```

## 使用建议

### 模板选择原则

1. **矛盾模板**：根据问题的特征选择最相关的模板
   - 唯一性矛盾：当现象具有独特性时
   - 反向矛盾：当需要质疑主流假设时
   - 时间矛盾：当现象具有时间维度时
   - 边界矛盾：当现象在特定条件下不成立时
   - 空间矛盾：当现象具有地域差异时
   - 群体矛盾：当现象在不同群体间差异明显时

2. **反事实推演模板**：根据风险评估需求选择
   - 因素移除：验证关键因素的重要性
   - 因果链：预测连锁反应
   - 极端边界：评估最坏情况
   - 时间反转：评估历史决策
   - 资源约束：制定备选方案

### 组合使用策略

**完整分析流程**：
1. 使用3-6种矛盾模板识别异常矛盾点
2. 使用假设竞争矩阵量化评估假设
3. 使用2-3种反事实推演模板验证假设

**快速分析流程**：
1. 使用1-2种矛盾模板识别核心矛盾
2. 使用假设竞争矩阵评估主要假设
3. 使用1种反事实推演模板快速验证
