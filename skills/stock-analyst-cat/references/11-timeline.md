## 实施时间表

### 周一盘前（8:00-9:00）

#### 执行顺序
1. **8:00-8:15** - 第0层市场全景扫描
   - 快速定位当日所有潜在热点
   - 搜索主流财经媒体、交易所公告
   - 输出：潜在热点清单

2. **8:15-8:30** - 第1层官方确认层
   - 验证热点的官方/权威信息源
   - 搜索国务院、证监会、发改委等官网
   - 输出：经过官方验证的热点清单

3. **8:30-8:45** - 第2层专业解读层
   - 获取券商研报、专业媒体分析
   - 理解热点的深层逻辑和影响
   - 输出：带有深度逻辑的热点清单

4. **8:45-9:00** - 建立逻辑链条与评分
   - 串联各层信息，建立因果关系
   - 执行验证流程，评估舆情热度
   - 综合评分并输出预警

### 周一早盘（9:30-10:00）

#### 执行顺序
1. **9:30-9:40** - 实时监控开盘异动
   - 观察开盘竞价情况
   - 识别开盘异动板块
   - 验证盘前预测准确性

2. **9:40-9:50** - 调整评分权重
   - 根据开盘情况调整评分
   - 更新热点优先级
   - 修正预测结论

3. **9:50-10:00** - 输出热点确认报告
   - 确认热点有效性
   - 更新态势评级
   - 输出确认报告

### 盘中持续（10:00-14:00）

#### 执行顺序
1. **10:00-11:30** - 跟踪热点演变
   - 监控热点板块走势
   - 捕捉新驱动因素
   - 验证逻辑持续性

2. **11:30-13:00** - 午盘休整
   - 复盘上午走势
   - 准备午后策略
   - 识别午后机会

3. **13:00-14:00** - 下午监控
   - 跟踪午后热点演变
   - 实时风险排查
   - 动态调整策略

### 收盘复盘（14:00-15:00）

#### 执行顺序
1. **14:00-14:30** - 复盘全天走势
   - 复盘全天热点走势
   - 验证预测准确性
   - 识别预测偏差

2. **14:30-14:45** - 识别次日机会
   - 识别次日潜在热点
   - 建立热点观察清单
   - 准备次日策略

3. **14:45-15:00** - 输出复盘报告
   - 输出复盘报告
   - 总结经验教训
   - 优化次日策略

## 实时监控与动态调整

### 盘中监控要点

#### 1. 热点演变监控
- 热点板块涨幅变化
- 涨停股数量变化
- 资金流向变化
- 板块联动效应

#### 2. 新驱动因素捕捉
- 突发消息
- 政策变化
- 业绩公告
- 资金异动

#### 3. 风险信号监控
- 资金出逃信号
- 情绪降温信号
- 逻辑证伪信号
- 监管风险信号

### 动态调整策略

#### 评分调整
- 根据实时信息调整评分
- 更新热点优先级
- 修正预测结论

#### 评级调整
- 根据最新情况更新评级
- 调整操作建议
- 更新风险提示

#### 策略调整
- 根据市场变化调整策略
- 修正作战地图
- 更新操作计划

## 智能日期识别机制

### 日期识别逻辑
```python
def 获取搜索日期():
    today = datetime.now()
    weekday = today.weekday()
    
    if weekday == 0:  # 周一
        # 搜索上周五 + 周末两天
        return {
            "主时间窗口": (today - timedelta(days=3)).strftime("%Y-%m-%d"),  # 上周五
            "补充时间窗口": [
                (today - timedelta(days=2)).strftime("%Y-%m-%d"),  # 周六
                (today - timedelta(days=1)).strftime("%Y-%m-%d"),  # 周日
            ],
            "实时时间窗口": today.strftime("%Y-%m-%d")  # 今日
        }
    else:
        # 搜索昨天 + 今日
        return {
            "主时间窗口": (today - timedelta(days=1)).strftime("%Y-%m-%d"),  # 昨天
            "补充时间窗口": [],
            "实时时间窗口": today.strftime("%Y-%m-%d")  # 今日
        }
```

### 时间窗口说明
> **重要**：时间窗口统一引用 01-core.md 双窗口策略
> - **预警窗口**：0-24小时（国际消息提前扫描，中英文并行）
> - **主窗口**：24-96小时（核心判断依据，完整评分）
> - **次窗口**：96小时-7天（补充参考，降权评分）
> - **历史窗口**：7天以上（**禁止作为核心依据**）

## 国际会议时效性判断标准（2026-03-25复盘新增）

> **核心原则**：国际会议催化是最强催化类型，必须精确判断时效性

### 【时效性判断规则】
| 会议阶段 | 判定标准 | 操作策略 |
|----------|----------|----------|
| **会前3日** | 开始追踪相关板块 | 标记为"潜在热点"，不评分 |
| **会议期间** | 实时跟踪催化内容 | 发布新产品=当日最强催化 |
| **会后3日** | 判断是否"利好兑现" | 超过3日=催化消退 |

### 【GTC大会特殊规则】
- 英伟达GTC大会期间：光通信/AI算力催化强度×1.5
- 会议期间发布新产品：自动纳入"当日最强催化"
- 会议结束后的解读行情：一般不超过3个交易日
- 超过3个交易日仍未启动：降权处理

### 【时间窗口校准规则】
| 报告类型 | 使用窗口 | 说明 |
|----------|----------|------|
| 盘前报告 | 会前催化 | 识别潜在热点 |
| 盘中报告 | 会中催化 | 跟踪实时催化 |
| 盘后报告 | 会后评估 | 判断催化是否延续 |
