# 📝 多平台文章生成器（支持今日头条/微信公众号）

自动生成符合平台调性的高质量原创文章，支持头条（信息流）和公众号（私域）两种模式

---

## 🎯 平台参数说明

**默认平台**：toutiao（今日头条）
**支持平台**：toutiao（今日头条）、wechat（微信公众号）

**使用方式**：
- 生成头条文章：直接说"写头条文章"或"根据热点报告写头条"
- 生成公众号文章：说"写公众号文章"或"根据热点报告写公众号文章"或加参数`--platform wechat`

**平台差异**：
| 维度 | 今日头条（toutiao） | 微信公众号（wechat） |
|------|-------------------|-------------------|
| 内容长度 | 800-1500字，短平快 | 1500-2500字，深度完整 |
| 用户心智 | 信息流刷内容，浅阅读 | 主动关注，深度阅读，信任度高 |
| 内容生命周期 | 24小时失效 | 永久沉淀，长尾效应强 |
| 标题风格 | 直白直接，信息明确 | 有价值点，有情绪共鸣 |
| 结尾要求 | 简单互动即可 | 必须有关注引导+风险提示 |
| 合规要求 | 简单风险提示 | 强化合规模块，符合财经监管 |

## 第4步：爆款分析

### 📌 辅助技能调用：social-content-generator-0.1.0
**调用时机**：分析爆款文章时（默认调用）
**调用方式**：读取 `../social-content-generator-0.1.0/SKILL.md`（相对于本技能目录）
**获取内容**：
- 爆款逆向工程6步框架（NICHE ID → SCRAPE → ANALYZE → PLAYBOOK → LAYER VOICE → CONVERT）
- 钩子公式（好奇心钩子、故事钩子、价值钩子、反直觉钩子、社会证明钩子）
- 平台策略（LinkedIn/Twitter/Instagram/TikTok/Facebook最佳实践）
- 内容再利用地图

### 分析范围
- 搜索今日头条平台该主题的爆款文章（最近3个月内）
- 筛选高阅读量，高互动量的文章
- 仅分析标题、结构、互动钩子和配图风格，不使用具体内容

### 爆款逆向工程6步框架（来自social-content-generator）

**Step 1: NICHE ID — 找到对标账号**
- 识别10-20个同领域高互动创作者
- 选择标准：持续发布、高互动率、受众重叠

**Step 2: SCRAPE — 收集爆款内容**
- 收集500-1000+篇帖子
- 数据：帖子内容、互动指标、发布时间、格式类型

**Step 3: ANALYZE — 提取有效模式**
- 按互动率排序，分析Top 10%
- 识别：钩子类型、内容长度、格式模式、发布时间

**Step 4: PLAYBOOK — 编写可复用模板**
- 钩子模式文档化
- 格式模板文档化
- CTA模式文档化

**Step 5: LAYER VOICE — 叠加个人风格**
- "聪明的朋友"语气
- 具体 > 模糊（用具体数字）
- 短句、呼吸感、重点独立

**Step 6: CONVERT — 转化为行动**
- 软转化：评论区链接、DM触发
- 硬转化：产品自然提及、案例展示

### 钩子公式应用（来自social-content-generator）

| 钩子类型 | 公式 | 示例 |
|----------|------|------|
| 好奇心钩子 | "我[意外行为]然后[意外结果]" | "我停止日更后互动翻倍了" |
| 故事钩子 | "[时间]前，[意外事发生了]" | "上周，我差点犯了个大错" |
| 价值钩子 | "如何[期望结果]（不[常见痛点]）" | "如何提高转化率（不烧钱）" |
| 反直觉钩子 | "不流行观点：[大胆声明]" | "不流行观点：SEO已死" |
| 社会证明钩子 | "我们在[时间]内[达成结果]" | "我们30天内涨粉1万" |

### 分析结果呈现

```

- 文章结构：4个章节，总字数1500-2000字

- 配图风格：封面+2-3张章节插图，真实照片风格
- 钩子模式：[从爆款逆向工程提取的具体模式]
```

### 写作规则
- 标题：20-30字，吸引点击但不标题党
- 开头：100-150字，快速切入主题
- 章节：3-5个，每个章节300-600字
- 语言：口语化、简洁有力，生动形象
- 数据：每个观点都有数据或案例支撑
- 互动钩子：自动添加3-4个（观点类）或2-3个（教程类）

### 互动钩子位置

- 开头：制造悬念或引发好奇
- 章节结尾：提出开放性问题
- 总结：邀请读者分享经验或观点

### 📌 去AI痕迹处理（来自humanizer-zh）

**写完文章后，必须执行去AI痕迹检查：**

**AI写作特征检测清单**：
- [ ] 夸大的象征意义（"象征着"、"代表着"）
- [ ] 宣传性语言（"在...指导下"、"深入贯彻"）
- [ ] 以-ing结尾的肤浅分析（"通过...实现...ing"）
- [ ] 模糊的归因（"有专家认为"、"众所周知"）
- [ ] 破折号过度使用
- [ ] 三段式法则机械化
- [ ] AI高频词汇（"综上所述"、"总而言之"、"值得注意的是"）
- [ ] 否定式排比（"不是...而是..."）
- [ ] 过多的连接性短语（"首先...其次...最后..."）

**修复规则**：
| AI特征 | 修复方式 |
|--------|----------|
| "象征着" | 删除或换成具体描述 |
| "在...指导下" | 删除，直接说事 |
| "有专家认为" | 具体点名或删除 |
| "综上所述" | 换成自然过渡 |
| "首先...其次..." | 用短句、列表替代 |
| 破折号过多 | 保留最多1个/段 |

**输出示例**：
```
✅ 去AI痕迹检查：
- 象征意义：无
- 官腔套话：无
- AI高频词：已替换3处
- 连接短语：已优化
- 文章自然度：高
```
