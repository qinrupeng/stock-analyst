2. 评分计算器

```

         ↓

         ↓
Step 4: 输出报告 → 格式化输出
```

## Step 1: 热点发现

**文件**: `hotspot-checklist.md`

### 执行步骤

1. 确定分析场景（盘前/盘中/盘后/个股）
2. 按6个层级逐层检查
3. 记录发现的热点
4. 初步评分筛选
5. 确定进入分析的热点（≥3个）

### 热点识别关键词

| 类型 | 关键词 |
|------|--------|
| 政策 | 国务院、部委、规划、意见 |
| 业绩 | 预增、增长、盈利、订单 |
| 技术 | 突破、量产、商用、首发 |
| 事件 | 并购、重组、激励、地缘 |
| 资金 | 净流入、涨停、换手 |
| 情绪 | 热搜、讨论、推荐 |

---

## Step 2: 深度分析

**文件**: `score-calculator.md`

### 执行步骤

1. 对每个达标热点（≥75分）进行分析
2. 逐项填写消息面评分
3. 逐项填写市场预期评分
4. 逐项填写情绪驱动评分
5. 填写加减分项
6. 计算总分
7. 确定评级

### 评分公式

```
总分 = 消息面驱动(45分) + 市场预期(25分) + 情绪驱动(30分) = 100分
```

> **风险分级**：风险分级（🟢低/🟡中/🟠高/🔴极高）独立评估，不计入热点评分

### 评级标准

| 评级 | 阈值 | 仓位建议 |
|------|------|----------|
| 核爆级 | ≥90分 | 30-50% |
| 沸腾级 | ≥75分 | 15-30% |
| 发酵级 | ≥50分 | 5-15% |
| 观望级 | ≥40分 | 0-5% |
| 冷却级 | <40分 | 0% |

---

## Step 3: 质量审查

**文件**: `review-checklist.md`

### 执行步骤

1. 对每个分析结果进行审查
2. 逐项检查7大检查点
3. 记录问题
4. 计算质量分
5. 给出审查结论

### 审查结论

| 结论 | 条件 |
|------|------|
| ✅ 通过 | 总分≥80，无致命问题 |
| ⚠️ 建议修改 | 无致命问题，有优化建议 |
| ❌ 需要修改 | 1-2个致命问题 |
