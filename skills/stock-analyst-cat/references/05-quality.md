# 质量控制检查清单

本文档定义喵娘分析技能的输出前专项质量检查标准。

> **重要**：本文档为质量检查的执行清单，引用关系如下：
> - **态势评级标准**：统一引用 02-scoring.md
> - **信源等级标准**：统一引用 06-sources.md
> - **风险等级标准**：统一引用 02-scoring.md

---

## 检查流程概述

每个热点主题报告在输出前，必须通过**全部专项质量检查**。

---

## 1. 态势评级专项检查

### 检查项
- [ ] **标准性**：是否使用了五种标准评级之一？
  - ⚡⚡⚡⚡⚡核爆级
  - ⚡⚡⚡⚡沸腾级
  - ⚡⚡⚡发酵级
  - ⚡⚡观望级
  - ⚠️高风险预警级
- [ ] **准确性**：评级是否符合知识库判定标准？
- [ ] **完整性**：态势评级是否包含三部分？
  - 知识库标准态势评级
  - 喵娘锐评（50字内精炼点评）
  - 风险等级（🟢/🟡/🟠/🔴）

---

## 2. 信源标注专项检查

> **重要**：信源等级定义统一引用 06-sources.md，以下为快速参考

### 检查项
- [ ] **标识标准**：是否使用标准标识？
  - [🟢🟢S1级] 一级信源（官方机构）
  - [🟢S2级] 二级信源（权威媒体）
  - [🔵A级] 三级信源（专业财经媒体）
  - [🟡B级] 四级信源（地方媒体/行业媒体）
  - [🟠C级] 五级信源（数据服务商）
- [ ] **格式完整**：是否包含完整格式？
  - `[信源等级标识] [YYYY-MM-DD]-信源名称：[信息精要] （对市场生态的冲击解读） {时效标识}`
- [ ] **冲击解读**：每个信源是否包含"对市场生态的冲击解读"？
- [ ] **时效标识**：每个信源是否有时效标识？
  - 🆕 实时（0-6小时）
  - 📅 当日（6-24小时）
  - ⏰ 隔日（24-72小时）
  - ❌ 过期（>72小时）
- [ ] **排序合规**：信源是否按等级+时间规则排序？
  - 等级排序：S1级→S2级→A级→B级→C级
  - 时间排序：同等级内时间倒序（最新优先）

### ⚠️ 因果链标注专项检查（2026-03-28新增）
> **详细规则**：参见 06-sources.md"因果链标注规则"章节

- [ ] **因果分离检查**：每个S1/S2/A级信息是否通过"因果链三问"？
  - 问1：这是"为什么涨"还是"涨了多少"？
  - 问2：如果没有这个事实，行情还会发生吗？（能→因，不能→果）
  - 问3：这是"原始驱动"还是"市场表现"？
- [ ] **降级检查**：以下错误标注是否已纠正？
  - ❌ 行情结果标注为S1/S2级 → 应降级为A级
  - ❌ "主力净流入X亿"标注为S1/S2级 → 应降级为A级（果）
  - ❌ "板块涨停潮"标注为S1/S2级 → 应降级为A级（果）
  - ❌ "股价突破X元"标注为S1/S2级 → 应降级为A级（果）
- [ ] **核心拼图资格检查**：只有"因"（驱动事实）才能作为S/A级核心拼图
  - 驱动事实（如禁令、政策、技术突破）→ ✅ 可作为S/A级
  - 行情结果（如涨停、资金、涨幅）→ ❌ 降级为A级，不能作S/A级

## 深度分析质量控制检查清单

### 溯因推理三件套检查

- [ ] **苏格拉底式提问检查**
  - [ ] 数量：≥3条
  - [ ] 覆盖：核心假设质疑、反例寻找、逻辑漏洞识别
  - [ ] 提问方式：开放式、递进式、反事实

- [ ] **正反方辩论检查**
  - [ ] 完整性：正方观点、反方观点、多空对撞、最终结论
  - [ ] 交锋性：观点有明确对撞，不是单边表述
  - [ ] 权衡性：有明确的概率评估和权衡判断

- [ ] **预演失败检查**
  - [ ] 数量：≥3个场景
  - [ ] 覆盖：政策风险、基本面风险、情绪风险
  - [ ] 完整性：每个场景包含触发阈值、应对方案、概率评估

- [ ] **动态调整机制检查**
  - [ ] 监控指标：基于预演失败的触发阈值设置
  - [ ] 调整规则：明确"如果X触发，则执行Y"的逻辑
  - [ ] 复盘机制：收盘后评估三件套执行效果

### 国际信源检查（新增）

- [ ] **国际A级媒体检查**
  - [ ] 是否扫描了Bloomberg、Reuters等国际A级媒体？
  - [ ] 国际信源是否有至少2个A+级信源交叉验证？
  - [ ] 是否标记了时间差（国际领先 vs 国内同步）？

- [ ] **A股供应商关联检查**
  - [ ] 是否识别了国际企业消息中的A股上市公司？
  - [ ] 是否验证了A股供应商的真实性（公告、财报）？

### 风险传导路径检查（新增）

- [ ] **风险传导路径推演**
  - [ ] 是否推演了"如果风险触发，传导路径是什么？"
  - [ ] 是否设置了风险触发阈值？
  - [ ] 是否提供了具体的应对方案？

## 步骤执行与质量控制集成检查清单

### 步骤1：周末消息面扫描
- [ ] 第0层市场全景扫描完成
- [ ] 第1层官方确认层完成
- [ ] 潜在热点清单输出

### 步骤2：建立逻辑链条
- [ ] 逻辑链条清晰完整
- [ ] 因果关系明确
- [ ] 符合市场运行逻辑

### 步骤3：验证市场预期
- [ ] 资金情绪验证完成
- [ ] 板块热度验证完成
- [ ] 市场关注度验证完成
- [ ] 国内外联合共振验证完成

### 步骤4：评估舆情热度
- [ ] 媒体报道数量评估
- [ ] 社交媒体讨论热度评估
- [ ] 投资者讨论活跃度评估

### 步骤5：综合评分
> **评分标准**：参见 02-scoring.md 第一层评分机制
- [ ] 是否已判断热点类型（政策驱动/技术突破/情绪驱动/资金驱动/概念炒作/国际事件）？
- [ ] 是否已选择对应评分模式（理性模式/量化模式/混合模式）？
- [ ] 调用四阶段动态评分机制（盘前/早盘/盘中/收盘阶段权重调整）
- [ ] 消息面驱动 + 市场预期 + 情绪驱动（比例因模式而异）
- [ ] 总分≥75分保留

## 7. 禁止事项清单 (20条)

**绝对禁止行为：**

1. ❌ 禁止只看好消息，忽视风险
2. ❌ 禁止不查资金流向就判断热点
3. ❌ 禁止只跟踪科技股，忽视周期股
4. ❌ 禁止不查外围市场就判断A股走势
5. ❌ 禁止使用个人笔记/自媒体信源作为核心依据
6. ❌ 禁止遗漏评分≥75分的热点
7. ❌ 禁止不按评分排序输出
8. ❌ 禁止在用户追问后才补充输出
9. ❌ 禁止跳过强制检查点验证
10. ❌ 禁止在执行确认清单未完成时输出
11. ❌ 禁止使用超出72小时窗口的信息作为核心判断依据
12. ❌ 禁止周一搜索"今日"数据（今日还未开盘）
13. ❌ 禁止跳过漏判防范检查清单
14. ❌ 禁止未执行多重驱动分析就输出热点
15. ❌ 禁止未按标准进行态势评级
16. ❌ 禁止隐藏态势评级依据
17. ❌ 禁止输出执行完整性状态不完整的报告
18. ❌ 禁止在盘前使用"当日资金"表述（盘前无当日数据）
19. ❌ 禁止只验证昨日资金，不验证资金趋势
20. ❌ 禁止忽略融资融券情绪

## 8. 必须执行清单 (20条)

**必须执行行为：**

1. ✅ 必须标注每个信息的信源等级和发布时间
2. ✅ 必须进行必要的交叉验证
3. ✅ 必须检查信源充足度
4. ✅ 必须检查是否存在信源冲突
5. ✅ 必须提示信源相关风险
6. ✅ 必须优先使用S级/A级信源
7. ✅ 必须对技术突破型热点进行专项搜索
8. ✅ 必须使用多轮搜索机制验证热点
9. ✅ 必须严格控制时间窗口，核心判断必须基于72小时内的信息
10. ✅ 必须在报告中标注时间窗口状态
11. ✅ 超出72小时的信息必须仅作为背景参考
12. ✅ 周一盘前只搜索上周五+周六+周日数据
13. ✅ 必须执行漏判防范检查清单
14. ✅ 必须执行多重驱动分析机制
15. ✅ 必须按标准进行态势评级
16. ✅ 必须展示态势评级依据
17. ✅ 必须使用智能关键词匹配搜索
18. ✅ 必须完成所有强制检查点验证
19. ✅ 必须完成执行确认清单勾选
20. ✅ 必须在输出头部显示执行完整性状态

## 遗漏热点检查清单（2026-03-25复盘新增）

> **核心原则**：复盘时必须强制执行遗漏检查，防止同类遗漏重复发生

### 【遗漏热点分类】
| 类型 | 定义 | 遗漏原因 |
|------|------|----------|
| A类遗漏 | 有催化但未识别 | 信源扫描不到位 |
| B类遗漏 | 有催化但低估 | 催化强度评估错误 |
| C类遗漏 | 完全未察觉 | 假设/逻辑框架有盲区 |

### 【遗漏热点复盘检查清单】
1. [ ] 今日涨停个股中，哪些不在盘前预测中？
2. [ ] 涨停板块的市场规模是否超过盘前预测主线？
3. [ ] 是否有国际会议/事件催化被忽视？（GTC、WWDC等）
4. [ ] 是否有"首次提出"的政策表述未解读？
5. [ ] 美股对应板块涨跌是否被追踪？
6. [ ] 是否有新主线形成信号？（连续3日涨停、7连板以上）
7. [ ] 遗漏热点是否在主窗口（24-96h）内被识别？
8. [ ] 遗漏热点下次如何避免？
9. [ ] 遗漏原因是否可归因于信源/逻辑/假设？
10. [ ] 是否已更新"遗漏热点特征库"？

### 【遗漏热点记录格式】
```
遗漏热点：[名称]
催化来源：[信源]
遗漏类型：[A/B/C类]
遗漏原因：[分析]
改进措施：[具体行动]
```
